# PROMPTS.md - AI Summary Generation Prompt Documentation

This document explains the design, evolution, and fallback strategies for the AI-generated audit summary prompt used in **JustCheckin**.

---

## 1. Production Prompt (Gemini 1.5 Flash)

This is the system prompt and instructions compiled for the Gemini model in `backend/server.js`:

### System/Role Framing
> "You are a blunt, numbers-first financial advisor for startups."

### Input Data
The audit results object generated by the `auditEngine.js` is serialized as JSON:
```json
{
  "perTool": [
    {
      "toolKey": "cursor",
      "toolName": "Cursor",
      "currentPlan": "Business",
      "currentSpend": 200,
      "recommendedPlan": "Pro",
      "recommendedSpend": 100,
      "monthlySavings": 100,
      "annualSavings": 1200,
      "flag": "overpaying",
      "reason": "Team of 5 on Business at $40/seat. Pro at $20/seat covers identical features for teams under 20. Switch saves $100/mo."
    }
  ],
  "totalMonthlySavings": 100,
  "totalAnnualSavings": 1200,
  "overallVerdict": "medium-savings",
  "redundancies": []
}
```

### Prompt Instructions & Constraints
```
Given this AI tool audit result, write a 90-110 word summary paragraph.
Rules:
- Start with the single biggest finding.
- Cite 2 specific dollar amounts from the data.
- End with one concrete next step.
- Do NOT use bullet points — flowing paragraph only.
- Do NOT be sycophantic ("Great job!" etc.).
```

---

## 2. Why We Wrote It This Way (Design Decisions)

* **"Blunt, numbers-first" Framing:** Prevents vague, flowery introductions (e.g., *"Looking at your audit, it seems like you have some interesting setups..."*). Startup founders appreciate directness and need immediate numbers.
* **Strict Word Count Constraint (90-110 words):** Ensures the summary fits perfectly in the results screen card without expanding or breaking the vertical grid layout.
* **"Cite 2 specific dollar amounts":** Forces the model to extract and display the exact savings (e.g., `$150/mo` or `$1,800/yr`) directly from the audit data instead of writing generic advice.
* **"Do NOT use bullet points":** The results page already contains a detailed bulleted list of tools and actions. The summary should be a readable paragraph card that looks like a human advisor wrote it.
* **"Do NOT be sycophantic":** A common LLM failure is opening with praise (*"Excellent job on keeping your Gemini costs low!"*). The advisor should be completely objective and business-oriented.

---

## 3. Prompt Iterations & What Didn't Work

### Iteration 1: The Helpful Assistant (Failed)
* **Prompt:** *"You are a helpful assistant. Review this AI audit and summarize how the company can save money."*
* **Why it failed:** The output was overly wordy, friendly, and lacked concrete metrics. It did not mention the exact monthly or annual savings figures, relying on generalities like *"You should switch to cheaper tiers."*

### Iteration 2: Bulleted List (Failed)
* **Prompt:** *"Write a bulleted summary of the core recommendations and estimated savings."*
* **Why it failed:** It duplicated the layout of the "Detailed Analysis by Tool" section located right below the hero card, resulting in a cluttered, repetitive user interface.

### Iteration 3: Unconstrained Length (Failed)
* **Prompt:** *"Explain the biggest savings opportunities and next steps in a paragraph."*
* **Why it failed:** The model occasionally generated 250+ words of text, pushing the visual layout below the fold and making it difficult for the user to screenshot the results screen easily.

## 4. Fallback Strategy

To guarantee a reliable, production-ready app, the system implements a rule-based fallback generator if the API key is invalid, missing, or rate-limited. This fallback runs entirely client/server-side in standard JavaScript without external dependencies:

```js
const savings = auditData?.totalMonthlySavings || 0;
summaryText = savings > 0
  ? `Your audit identified $${savings}/month in potential savings across your AI tool stack. The largest opportunity is switching over-provisioned team plans to individual tiers — a common pattern for teams under 10. Redundant tools are also costing you more than necessary. Start by cancelling the lowest-value subscription and reallocating that budget to the tools your team uses daily.`
  : `Your AI tool spend looks well-optimized for your team size and use case. You're on appropriately sized plans with no obvious redundancies. Keep an eye on seat counts as your team grows — the transition from individual to team tiers can catch you guard if you add headcount quickly.`;
```
This ensures that the user **always** gets a personalized summary with accurate numbers, maintaining the premium feel under any network condition.
